## 🎯 简化更新方案

考虑到部分工具文件复杂度较高（如六合一提成计算、奖金计算等），我们采用**渐进式更新策略**：

### Level 1: 仅更新头部和导航（适用于复杂工具）
```html
<!-- 1. 在 <head> 中添加样式引用 -->
<link rel="stylesheet" href="../common-styles.css">

<!-- 2. 在现有样式前添加，保持原有功能样式 -->
<style>
  /* 保留原有的功能性样式 */
</style>

<!-- 3. 只更新 header 部分 -->
<div class="tool-container">
  <header class="tool-header">
    <a href="../index.html" class="back-nav">← 返回首页</a>
    <h1>工具标题</h1>
    <p class="tool-subtitle">工具描述</p>
  </header>
  
  <!-- 保留原有的主体内容不变 -->
  <div class="content">
    <!-- 原有内容 -->
  </div>
</div>
```

### Level 2: 全面更新（适用于简单工具）
按照《样式统一指南.md》完整更新所有组件

### Level 3: 自定义优化（可选）
在统一样式基础上，针对特殊需求添加局部样式覆盖

## 📌 关键注意事项

### 1. CSS 优先级
新样式文件应该在原有样式**之前**引入：
```html
<link rel="stylesheet" href="../common-styles.css">
<style>
  /* 原有的功能样式（会覆盖 common-styles） */
</style>
```

### 2. 保留 JavaScript 功能
- 不要修改任何 JavaScript 代码
- 不要修改元素的 `id` 属性
- 可以添加 `class`，但不要删除原有 `class`

### 3. 渐进式测试
每更新一个文件后：
1. 在浏览器中打开测试
2. 检查基本功能是否正常
3. 检查样式是否符合预期
4. 有问题及时回滚
